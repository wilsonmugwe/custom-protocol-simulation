## Expected vs Observed

| Case            | Expected (theory)                                            | Observed (Wireshark & CSV) | Notes |
|-----------------|---------------------------------------------------------------|-----------------------------|-------|
| Handshake       | `seq=1` REQUEST “ClientHello” then `seq=1` ACK “ServerHelloAck” |                             |       |
| Normal flow     | `seq` increases by +1; paired REQUEST/RESPONSE; final BYE    |                             |       |
| Truncated       | `DECODE_FAIL:*` in analyzer; payload not parsed               |                             |       |
| Tampered tail   | `LEN_MISMATCH` and/or CRC integrity issue                     |                             |       |

| Bad version     | Decode error or ignored per decoder rules                     |                             |       |
| Baseline (TCP)  | 3-way handshake, HTTP GET/200 OK, valid ACK numbers          |                             |       |
| Baseline (DNS)  | Query/Answer share TXID; A record for example.com            |                             |       |
| Baseline (ICMP) | Echo request/reply share id/seq                               |                             |       |

## Reporting & Validation

### Report (per PCAP)
```powershell
mkdir reports

python custool.py report --enc proto ^
  --pcap captures.pcap/mixed_proto.pcap ^
  --json reports/mixed_proto.json ^
  --csv  reports/mixed_proto.csv ^
  --html reports/mixed_proto.html

python custool.py report --enc asn1 ^
  --pcap captures.pcap/errors_asn1.pcap ^
  --json reports/errors_asn1.json ^
  --csv  reports/errors_asn1.csv ^
  --html reports/errors_asn1.html
```

- CSV columns: `frame,decoded,seq,len,issues,error,payload_utf8`
- HTML shows **Total frames**, **Decoded**, **Frames with issues/errors**.

### Validate (PASS/FAIL)
```powershell
python custool.py validate --enc proto --pcap captures.pcap/normal_proto.pcap
python custool.py validate --enc asn1  --pcap captures.pcap/normal_asn1.pcap
```

### Manifest (integrity)
```powershell
python custool.py manifest captures.pcap/*.pcap > reports/manifest.json
```

## Wireshark Workflow & Filters

### Custom Protocol (UDP, default port 5555)
- Filters:
  - `udp.port == 5555`
  - `udp && frame.len < 20` (spot tiny malformed frames)
- Observe:
  - Handshake: two frames with `seq=1` (REQUEST then ACK)
  - Monotonic `seq` in **normal** captures
  - Where `DECODE_FAIL` / `LEN_MISMATCH` / `SEQ_NON_MONOTONIC` appear (cross-check CSV/HTML)

### BaselineNormal (Standard Protocols)
- **TCP/HTTP**
  - Filter: `tcp.stream eq 0` (or stream with `dport == 80`)
  - Confirm SYN → SYN/ACK → ACK; show `GET /index.html` and `HTTP/1.1 200 OK`
- **DNS**
  - Filter: `dns`
  - Verify TXID matches Query/Answer; `example.com A 93.184.216.34`
- **ICMP**
  - Filter: `icmp`
  - Show echo request & reply with same `id=0x1234, seq=1`

## Troubleshooting

- **Nothing decodes:** Ensure encoder flag matches file: `--enc proto` for `*_proto.pcap`, `--enc asn1` for `*_asn1.pcap`.
- **Filter empty:** Clear filters; confirm file contents first. For custom protocol, try `udp.port == 5555`.
- **CSV missing data:** Re-run `report` and open the **CSV**, not the JSON.
- **PASS/FAIL confusion:** `validate` fails if any frame has `issues` or `DECODE_FAIL`—expected for `errors_*` and often for `mixed_*`.
